# DINO-MM 下游任务全景报告

> 汇总：预训练迁移策略 + 完整任务矩阵 + Baseline 落地状态（截至 2026-03-15）

---

## 1. 目标

本文档总结 DINO-MM 预训练迁移到下游任务的全部工程进展，重点回答两个问题：

- **公平主表**：我们的预训练初始化是否稳定优于通用初始化 / 遥感初始化；
- **外部 benchmark**：接入强任务框架后整体性能能否逼近或超过 RingMo-Aerial / SkySense 系列。

## 2. 评估逻辑

### 2.1 公平主表

固定下游框架，仅替换初始化（`random` / `ImageNet` / `RSFM` / `DINO-MM` / `DINO-MM+PATC`）。

### 2.2 外部 benchmark

使用强框架头（如 `Cascade R-CNN`、MMTracking ByteTrack、强检索头）与公开方法对比。

> 两类表必须分开汇报，避免"框架增益"和"预训练增益"混淆。

## 3. 对齐 RingMo-Aerial / SkySense 的任务视角

RingMo-Aerial 与 SkySense 系列覆盖的任务远超单一检测或分割。结合四篇论文（RingMo-Aerial、SkySense、SkySense++、SkySense V2），本项目按任务分为三层：

- **已落地可跑**：分割、检测、旋转检测、跟踪、检索、场景分类、变化检测、三维重建；
- **论文覆盖但待补**：多模态时空映射任务；
- **我们可强化的独有方向**：跨视角大视差、缺失模态鲁棒、多模态一致性任务。

## 4. 本轮已完成工作（2026-03-15）

### 4.1 分割

- 默认头升级为仓库内 `UPerNet`，保留 `Lite-UPer` 作为旧基线；
- 增加 auxiliary supervision，训练路径接近标准强分割配置；
- 无需编译版 `mmcv` 即可训练/验证。

### 4.2 检测

- 框架选择支持：`fasterrcnn` / `fcos` / `cascade_rcnn`；
- 已实现 `DINO-MM -> MMDetection` 适配层；
- `cascade_rcnn` 现可通过 MMDetection 2.x 路径直接训练与评测。

### 4.3 旋转检测

- 新增 `train_oriented_detection.py`（MMRotate Oriented R-CNN 训练入口）；
- 已实现 `DINO-MM -> MMRotate` backbone 适配；
- 支持训练/推理前向与结果回转（旋转框输出 + 水平框评测兼容）。

### 4.4 跟踪

- 保留本地 `iou`、`bytetrack`；
- 新增 `mm_bytetrack`，接入 MMTracking 官方 ByteTracker 关联逻辑；
- 评测入口支持按参数切换关联器，便于公平对比。

### 4.5 检索

- 在原 `eval_retrieval.py` 基础上新增 `train_retrieval.py`；
- 支持跨模态/跨视角检索双塔对比学习微调；
- 支持数据集：`DroneVehicle`、`LLVIP`、`SUES-200`、`CVOGL`。

### 4.6 场景分类

- `train_classification.py` 已支持 `AID` / `RESISC-45` / `imagefolder`；
- 支持按 `--train_ratio` / `--val_ratio` 在无官方划分目录时做可复现分层划分；
- 可用于 RingMo-Aerial 论文口径下的线性探针对比。

### 4.7 变化检测

- 新增 `train_change_detection.py`、`eval_change_detection.py`；
- 新增 DINO-MM + BIT-style 双时相变化检测头（`downstream/change_detection.py`）；
- 支持 `LEVIR-CD` / `CDD` / `OSCD`（BIT_CD 风格目录）训练与评测。

## 10. 当前结论

截至 2026-03-17，DINO-MM 下游栈已完成从"轻量验证链路"到"强框架 SOTA 级可落地链路"的全面升级：

- 分割、检测、旋转检测、跟踪、检索、场景分类、变化检测、三维重建 **8 项任务全部具备可运行训练入口**；
- 所有训练脚本已升级为 SOTA 级配置：poly/cosine LR + warmup + 梯度裁剪 + backbone/head 分离学习率 + AMP；
- 变化检测升级为多尺度 FPN 融合 + 深度监督，三维重建完成 DINO-MM + Cas-MVSNet 全链路接入；
- `Cascade R-CNN`、MMTracking ByteTrack、Oriented R-CNN 适配已打通；
- 所有 baseline 仓库已 clone，OpenMMLab 全家桶（mmdet/mmseg/mmrotate/mmtrack）环境验证通过；
- **下一步核心瓶颈转为"全任务跑表与 baseline 数值对齐"**，而不是"框架本身不可用"。
